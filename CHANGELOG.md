# Changelog

## [0.7.2] - 2026-05-11

### Added

- Можливість видалити файл терміналів мерчанта в кешбеці при редагуванні
- Можливість повторної обробки транзакцій через розділ «Мерчанти»: кнопка «Перерахувати кешбек» у блоці редагування терміналів активується при додаванні нових терміналів
- Логіка автоматичного перерахунку транзакцій за поточний місяць при додаванні нових терміналів до мерчанта

![Демонстрація перерахунку кешбеку по мерчанту](images/changelog/all_types_recount.gif)

### Changed

- Оновлено версію Node.js
- Змінено форму мерчанта: розділено на дві вкладки — «Налаштування мерчанта» та «Налаштування терміналів»; додано пошук по назві мерчанта та окремі іконки для редагування налаштувань і терміналів
- Прибрано фільтр по типу карти на всіх роутах отримання даних по кешбеку в мобільному гейтвеї (крім кобрендингу)
- Змінено сторінку маппера по кафці/вебхуки: оновлено логіку перевірки маппера та відображення результатів
- Нове поле ручного додавання термінала тепер відображається вгорі при додаванні більше одного термінала
- Змінено назву блоку «Додати вручну» на «Додати термінали вручну» у формі мерчанта
- Кнопка «Перерахувати кешбек» тепер активна для нових мерчантів (без прив'язки до активного кешбеку)

### Fixed

- Після закінчення таймауту сесії відображалась сторінка Unauthorized Access замість сторінки логіну
- Не відображався тип операції в деталях операції в аналітиці по користувачу
- Відсутня можливість видалити файл терміналів мерчанта в кешбеці при редагуванні мерчантів
- Автоматично проставлявся null у полях name і location при додаванні терміналів до мерчанта файлом
- Обрізалась пагінація при редагуванні терміналів у розділі «Мерчанти»
- Невірний пошук по ID термінала через рядок пошуку в лістингу мерчантів
- Обрізалось нижнє виділення активної сторінки перегляду терміналів мерчанта
- Помилка при додаванні раніше експортованого файлу терміналів мерчанта
- Обрізалось поле вибраного термінала для архівації
- Розмір тултіпу не відповідав довжині тексту в ньому у налаштуваннях кешбеку
- Заголовок і опис сторінки першого налаштування курсів валют не були перекладені; поля «Валюта для конвертації» та «API для отримання курсів» відображались з різною висотою
- На формі додавання акційного реферального посилання відображався неклікабельний текст «Додати нове посилання»

## [0.7.1] - 2026-04-29

### Fixed

- Не працювала кнопка збереження загальних налаштувань дефолтного кешбеку: після редагування налаштувань та натискання «Зберегти» запит на збереження не виконувався

## [0.6.1] - 2026-04-27

### Added

- Фіксація в окрему таблицю фактів отримання доступних кешбеків по користувачу (перегляд без вибору) з налаштуванням вебхука
- Сегментація клієнтів по мерчанту в дефолтному кешбеку: можливість задати окремий відсоток кешбеку для сегменту та керувати доступністю дефолтних винагород для клієнтів поза сегментами
- Можливість завантажити приклад файлу-шаблона терміналів для налаштування сегментації
- Кнопка експорту списку клієнтів сегменту мерчанта
- Поле `transactionId` в імпорті транзакцій
- Налаштування запуску e2e тестів у рамках CI
- Режим тестування: можливість додавати кешбеки на різні (у т.ч. минулі) періоди через ENV-параметр

### Changed

- Логіка керування сегментами винесена виключно в дефолтний кешбек (прибрано дублювання в персональному)
- Імпорт транзакцій: додано підтримку поля `cardType`
- Перехід з `TransactionProcessStatus` на `TransactionCashbackDecision`
- Уточнена логіка максимальної суми кешбеку по мерчанту
- Змінено сторінку «Умови нарахування» та розділ «Умови виведення кешбеку»
- Уніфіковано формати полів по всіх формах системи
- Змінено назву архівованих параметрів у всіх меню
- Уточнені налаштування вебхуків
- Налаштування мерчантів: правки до відображення та логіки

### Fixed

- Кешбек не нараховувався для клієнта з сегменту мерчанта при дотриманні всіх умов дефолтного кешбеку
- Подвійне нарахування кешбеку по групі; відсутній пріоритет персонального кешбеку над дефолтним
- Відсоток кешбеку групи не додавався до основного нарахування при збігу MCC-коду
- Дефолтний кешбек з заповненим полем `Merchant Name` нараховувався без урахування назви мерчанта в транзакції по сегменту
- Не визначався мерчант по терміналу при обробці транзакції
- Не визначався мерчант по терміналу з невірним `merchantName`
- Не визначався мерчант по терміналу в розділі «Мерчанти»
- Система не визначала мерчанта для валідної транзакції в аналітиці
- Не нараховувався дефолтний кешбек мерчанта по терміналу в кешбеці
- Бекенд пропускав вибір мерчанта для клієнта, не включеного до дефолтного кешбеку
- Не відображалася назва мерчанта в лістингу транзакцій аналітики при реверсі
- Не визначалась група/мерчант при недостатній мінімальній сумі транзакції для нарахування
- GET `/api/v2/cashback/positions` не повертав кешбеки по сегментах для клієнта
- POST `/api/co-branded-cashback/customer` повертав 400 попри наявність активного кобрендингового кешбеку
- Кобренд. Веб API повертав порожній масив вибраних груп і мерчантів

## [0.6.0] - 2026-03-25

### Added

- Підтримка мультивалютності: довідник курсів валют (НБУ), автоматична конвертація транзакцій у системну валюту, збереження даних конвертації (оригінальна сума, валюта, курс) разом із транзакцією
- Окрема таблиця `INVALID_TRANSACTIONS` для невалідних транзакцій (відсутня валюта, невідомий курс тощо) з відображенням причини в аналітиці по користувачу
- Новий тип кешбеку — кобрендинговий: підтримка налаштувань по типу карти, паралельна робота з дефолтним кешбеком, окремі мобільні ендпоінти для отримання налаштувань та позицій
- Поле `Merchant Name` як додаткова умова матчингу транзакцій по мерчанту (разом з Terminal ID)
- Додаткові поля `terminal_name` та `terminal_location` при додаванні терміналів по мерчанту
- Лічильник загальної кількості завантажених терміналів для кожного мерчанта в розрізі всіх типів кешбеку
- Ендпоінт отримання даних по кешбеку для конкретної транзакції (пошук за ID транзакції)
- Пагінований ендпоінт отримання повної історії кешбеку користувача (продубльований на мобільний gateway)
- Поле `bannerUrl` в мобільному ендпоінті `/cashbacks/positions` (тільки для позицій типу мерчант)
- Поле `accountId` в `CASHBACK_HISTORY` — зберігається рахунок, на який здійснено виведення коштів
- Повернення `cashbackHistoryId` у відповіді ендпоінту `/withdraw` (для підтримки операцій реверсу)
- [Deeployalty] Відправка webhook про вибраний кешбек користувача на Deeployalty при зміні налаштувань кастомера
- Розгортання модуля кешбеків для клієнта «Кліринговий дім»
- Відображення помітки Deeployalty на мерчантах при їх виборі під час створення кешбеку
- Можливість перегляду помилок по рядках при імпорті транзакцій
- Відображення обов'язкових полів у всіх формах системи

### Changed

- [Deeployalty] Оновлено логіку синхронізації терміналів: термінали в розділі «Мерчанти» тільки доповнюються (не видаляються), в блоці мерчанта кешбеку — актуальні термінали поточної кампанії
- Аналітика на фронтенді відображає дати транзакцій у локальному часі (замість UTC)
- Поле `accountId` в ендпоінті `/cashback-histories/withdraw` стало обов'язковим
- Приховано сторінку статистики (дані були захардкоджені, не відображали реальних значень)
- Уніфіковано пагінацію на всіх вкладках з табличним відображенням

### Fixed

- Неможливо змінити налаштування мерчанта Deeployalty для кешбеку на наступний місяць
- Не мінусувався кешбек при рефанді з конвертацією валюти
- 500 помилка при виведенні кешбеку
- 500 помилка при виведенні коштів у клієнта «Кліринговий дім»
- Помилка авторизації на сервері Ощадбанку (дев)
- Помилка підключення бд (дев Ощадбанк)
- Нарахування кобрендингового кешбеку відбувалось без урахування типу картки
- Активний кобрендинговий кешбек на поточний місяць доступний до редагування
- Відсутня можливість вимкнути конвертацію валют при початковому налаштуванні системи
- Кнопка «Зберегти» була активна навіть без змін у налаштуваннях API валют
- UAH-транзакція проходила лише у верхньому регістрі (UPPERCASE)
- Oracle: обмеження `IN()` на 1000 елементів — виправлено обробкою батчами
- Після виходу з облікового запису відбувався редирект на `/setup-database` замість `/login`
- Відсутній редирект на сторінку входу після підтвердження виходу в розділі Settings
- Назви колонок `CreatedAt`, `Parent ID`, `LDAP Filter` не були перекладені українською у відповідних розділах
- Назви валют у розділі «Курс валют» відображались без назви країни в українській локалізації
- Сума в списку транзакцій відображалась без конвертації у системну валюту
- Кнопка «Документація» відкривала головну сторінку замість документації
- Файл терміналів по мерчанту вивантажувався порожнім
- Лічильник терміналів у картці мерчанта враховував термінали з кешбек-налаштувань
- Помилки при роботі з WebPlatform API (Кліринговий дім)
